## 🎬 功能演示场景

### 场景1: 创建一个完整的监控

**步骤**:
1. 点击"系统设置" → 找到"监控对象管理"区块
2. 点击"新建"按钮 → 表单所有字段清空
3. **基本配置填写**:
   - 名称: `小米品牌舆情监控`
   - 分组: 选择或新建"品牌监控组"
   - 标签: `品牌,舆情,热点`

4. **采集配置填写**:
   - 平台: 输入 `weibo,zhihu,baidu`（多个用逗号分隔）
   - 采集量: `60`（默认值）
   - 关键词: 
     ```
     小米
     小米手机
     高通骁龙
     MIUI系统
     ```
     * 每行一个或用逗号分隔 *

5. **执行配置设置**:
   - 监控间隔: `1800` (30分钟)
   - ✓ 勾选"启用监控" (使其自动运行)
   - 报告格式: 勾选 HTML + DOCX + PDF

6. **告警阈值设置**:
   - 负面占比: `0.3` (30%)
   - 风险分: `50` (中等)
   - 采集量阈值: `30`

7. 点击"保存设置" → 监控创建成功

**验证**:
- 列表中出现新监控
- 启用指示符显示 `●` (绿点)
- 最后执行时间显示 '未执行'

### 场景2: 编辑并禁用一个监控

**步骤**:
1. 从列表中点击刚创建的监控
2. 表单自动加载所有字段
3. 修改名称: `小米品牌舆情监控（暂停中）`
4. 修改执行配置:
   - ☐ 取消勾选"启用监控"（禁用）
   - 只保留 DOCX 格式
5. 修改告警阈值:
   - 负面占比: `0.5` (50%)
6. 点击"保存设置"

**验证**:
- 列表中监控的启用指示符变为 `○` (灰点)
- 报告格式只显示 DOCX

### 场景3: 表单验证演示

**测试缺少关键词**:
1. 点击"新建"清空表单
2. 只填写名称: `测试监控`
3. 选择平台: `weibo`
4. 不填关键词，直接点"保存设置"
5. 弹出警告: `请填写至少一个关键词`

**测试缺少报告格式**:
1. 填写所有必填项：名称、关键词、平台
2. 执行配置中不勾选任何报告格式
3. 点"保存设置"
4. 弹出警告: `请至少选择一个报告格式`

### 场景4: 立即执行并查看历史

**步骤**:
1. 选择一个监控
2. 点击"立即执行"按钮（绿色）
3. 等待1-2秒
4. 表单刷新，"执行历史"区域更新：
   ```
   最后执行: 2026-03-10 14:35:12
   执行状态: success (绿色)
   生成报告ID: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
   ```

**观察**:
- 即使监控被禁用，也可以手动执行
- 执行后自动加载最新的执行数据
- 执行状态用颜色表示（绿=成功，黄=运行中，红=失败）

---

### 场景5: 多分组管理

**步骤**:
1. 在"监控分组"表格中新建多个分组：
   - 品牌监控 (颜色:#2e8cff)
   - 竞品监控 (颜色:#ff6b6b)
   - 热点监控 (颜色:#ffc93d)

2. 为每个分组创建2-3个监控

3. 使用"监控对象管理"上方的分组过滤器：
   - 选择"品牌监控" → 只显示该分组的监控
   - 选择"全部分组" → 显示所有监控

## 🔍 常见问题排查

### Q: 新建后表单不能清空
**A**: 确保已点击"新建"按钮（而非编辑某个监控后清空）

### Q: 保存时说缺少报告格式
**A**: 确保在报告格式区域勾选了至少一个格式（HTML/DOCX/PDF）

### Q: 禁用的监控不显示在启用状态
**A**: 查看列表中的启用指示符，禁用的监控显示为灰色 `○`

### Q: 编辑后的内容没有保存
**A**: 检查console输出，确认API返回200状态码

### Q: 立即执行后看不到执行历史
**A**: 执行需要1-2秒，稍等后点击监控重新加载表单

## 💡 最佳实践

1. **监控命名**: 使用明确的命名，如 `品牌+监控类型+日期`
   - ✓ 好: `小米品牌舆情监控-2026年`
   - ✗ 差: `monitor1`, `test`

2. **关键词设置**: 
   - 不要太宽泛（会采集太多无关内容）
   - 不要太狭隘（可能采集不到有意义的内容）
   - ✓ 推荐: 3-10个相关关键词

3. **采集间隔设置**:
   - 热点话题: 300-600秒（5-10分钟）
   - 普通监控: 1800-3600秒（30分钟-1小时）
   - ✗ 不要太短: 会对数据源造成压力

4. **阈值配置**:
   - 保守(容易告警): 负面占比0.2, 风险分40
   - 中等(默认): 负面占比0.3, 风险分50
   - 严格(难告警): 负面占比0.5, 风险分70

5. **报告格式选择**:
   - HTML: 网页查看，实时交互
   - DOCX: 文件共享，编辑修改
   - PDF: 存档、发送，格式固定
